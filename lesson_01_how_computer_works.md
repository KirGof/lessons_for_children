# Урок 1 — Как устроен компьютер и интернет

## Зачем это нужно

Прежде чем писать код или настраивать серверы — нужно понять на чём всё это работает. Инженер который не понимает как данные идут от браузера до сервера и обратно — это механик который не знает как работает двигатель. Можно менять детали по инструкции, но при нестандартной поломке будешь беспомощен.

---

## Часть 1 — Как устроен компьютер

### Что изучить

**Железо и его роль:**
- **CPU (процессор)** — мозг компьютера. Выполняет инструкции: сложить числа, сравнить значения, перейти к другой инструкции. Работает с тактовой частотой (GHz = миллиарды операций в секунду). Ядра — это отдельные процессоры внутри одного чипа.
- **RAM (оперативная память)** — рабочий стол. Быстрая, но временная. Когда выключаешь компьютер — всё стирается. Программы живут в RAM пока работают.
- **Диск (HDD/SSD)** — шкаф с файлами. Медленнее RAM, но постоянный. SSD быстрее HDD потому что нет механических частей.
- **Сеть (NIC)** — почтовый ящик. Отправляет и получает данные по кабелю или Wi-Fi.

**Как это всё связано:**
Программа хранится на диске → ты её запускаешь → ОС загружает её в RAM → CPU читает инструкции из RAM и выполняет → результат записывается обратно в RAM или на диск.

**Операционная система (ОС):**
Linux, Windows, macOS — это программы которые управляют железом и дают другим программам к нему доступ. ОС — это посредник между твоим кодом и железом. Без ОС каждой программе пришлось бы самой разбираться как работать с конкретным диском или сетевой картой.

**Процессы и потоки:**
- **Процесс** — запущенная программа. У каждого процесса своя память, свой PID (идентификатор).
- **Поток** — задача внутри процесса. Один процесс может делать несколько вещей одновременно через потоки.

## Часть 2 — Как работает интернет

### Что изучить

**IP-адрес:**
Каждое устройство в сети имеет адрес — IP. Как почтовый адрес, только для компьютеров. Выглядит так: `192.168.1.1` (IPv4) или `2001:db8::1` (IPv6). IPv4 заканчиваются — поэтому переходят на IPv6.

**Клиент и сервер:**
- **Клиент** — тот кто запрашивает. Браузер, мобильное приложение, curl.
- **Сервер** — тот кто отвечает. Компьютер который слушает запросы и отдаёт данные.
- Это роли, не железо. Один и тот же компьютер может быть и клиентом и сервером.

**Порты:**
IP-адрес — это адрес здания. Порт — номер квартиры. На одном компьютере может работать много сервисов одновременно: веб-сервер на порту `80` (HTTP) или `443` (HTTPS), SSH на порту `22`, база данных на `5432`.

**DNS (Domain Name System):**
Люди запоминают имена (`google.com`), компьютеры работают с IP-адресами (`142.250.185.46`). DNS — это телефонный справочник интернета. Когда ты вводишь `google.com`, твой компьютер спрашивает DNS-сервер: "какой IP у google.com?" — и получает ответ.

**Что происходит когда ты открываешь сайт:**
1. Ты вводишь `https://github.com` в браузере
2. Браузер спрашивает DNS: "какой IP у github.com?"
3. DNS отвечает: `140.82.121.4`
4. Браузер устанавливает TCP-соединение с этим IP на порт `443`
5. Происходит TLS handshake (шифрование)
6. Браузер отправляет HTTP запрос: `GET / HTTP/1.1`
7. Сервер отвечает HTML-кодом страницы
8. Браузер рендерит страницу

**HTTP — протокол общения:**
Запрос от клиента выглядит так:
```
GET /index.html HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
```

Ответ от сервера:
```
HTTP/1.1 200 OK
Content-Type: text/html

<html>...</html>
```

Коды ответов:
- `200` — OK, всё хорошо
- `404` — Not Found, страница не найдена
- `500` — Internal Server Error, сервер сломался
- `301` — Redirect, страница переехала

**TCP/IP — как данные путешествуют:**
Данные разбиваются на маленькие кусочки — **пакеты**. Каждый пакет идёт своим путём через маршрутизаторы (роутеры) и собирается на месте назначения. TCP гарантирует что все пакеты дошли и в правильном порядке.

## Практические задания

### Задание 1.1 — Изучи своё железо
Открой терминал Ubuntu и выполни команды:
```bash
# Информация о процессоре
lscpu

# Информация об оперативной памяти
free -h

# Информация о дисках
df -h

# Все запущенные процессы
ps aux
```

**Что нужно сделать:** Запиши в заметки — сколько ядер у процессора, сколько RAM, сколько места на диске. Объясни своими словами что делает каждая команда.

---

### Задание 1.2 — Проследи за DNS
```bash
# Установи инструменты если нет
sudo apt install dnsutils -y

# Узнай IP адрес сайта
nslookup google.com

# Подробный DNS запрос
dig google.com

# Посмотри свой IP
curl ifconfig.me
```

**Что нужно сделать:** Найди IP-адреса для `github.com`, `google.com`, `ya.ru`. Напиши — они одинаковые? Почему или почему нет?

---

### Задание 1.3 — Посмотри как работает HTTP
```bash
# Установи curl если нет
sudo apt install curl -y

# Сделай HTTP запрос и посмотри заголовки
curl -v https://example.com 2>&1 | head -50

# Посмотри только заголовки ответа
curl -I https://github.com
```

**Что нужно сделать:** Сделай запрос к `https://httpbin.org/get` — это специальный сайт для тестирования HTTP. Найди в ответе свой IP-адрес. Объясни что означает каждый заголовок который вернул сервер.

---

### Задание 1.4 — Проследи путь пакета
```bash
# Установи traceroute
sudo apt install traceroute -y

# Посмотри через сколько узлов идёт пакет до google.com
traceroute google.com

# Проверь соединение
ping -c 5 google.com
```

**Что нужно сделать:** Запусти `traceroute google.com`. Посчитай сколько "прыжков" (hops) делает пакет. Нарисуй схему (можно от руки) — как пакет идёт от твоего компьютера до Google.
