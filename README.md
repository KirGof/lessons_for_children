# Курс DevOps/Backend — Индекс и план прохождения

## С чего начать

Если ты только открыл этот курс — читай этот файл полностью.
Здесь: структура, сколько времени займёт, в каком порядке, и что делать если застрял.

## Структура курса

**25 уроков, разбитых на 4 уровня.**
Каждый урок = теория + практические задания + финальный проект.
Финальный проект каждого урока — коммит с тегом версии в репозиторий `todo-service`.

## Как работать с уроком

Каждый урок устроен одинаково:

```
1. Прочитай теорию целиком (не пропускай код — читай)
2. Воспроизведи примеры на своей машине
3. Выполни задания 1–4 по порядку
4. Задание 5 (финальный проект) — только после 1–4
5. Коммит с тегом версии
6. Переходи к следующему уроку
```

**Не пропускай задания.** Можно не делать задание 5 если времени мало — но 1–4 обязательно.

## Если застрял

**Ошибка в коде:** прочитай ошибку полностью, загугли точный текст ошибки.

**Не понимаю концепцию:** открой официальную документацию (ссылки в конце каждого урока), прочитай главу.

**Застрял больше часа:** это нормально. Запиши вопрос конкретно ("функция X возвращает ошибку Y при условии Z") — конкретный вопрос получает конкретный ответ.

**Скучно / слишком просто:** пропусти теорию, сразу к заданиям. Если задания тоже просты — переходи к следующему уроку.
